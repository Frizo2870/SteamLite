Intermediate_Concepts:

En esta seccion podra encontrar los distintos metodos y conocimientos de nivel intermedio y con estos poder manipular las consultas de determinada manera, por lo que vera en esta seccion sera en relacion con JOINS y con FUNCIONES, las cuales le permitiran hacer mas facil consultas de su interes.

Tambien le quisieramos mostrarle graficas en las cuales le permitira comprender de manera mas sencilla que es lo que se esta ejecutando. Dichas graficas van a estar elaboradas por medio del lenguaje de programacion de Python y por medio de la libreria Plotly se generara el grafico.

Quisieramos que al final de esta seccion se pueda elaborar consultas realizadas por medio de JOINS o invocando funciones con el fin de realizar consultas de interes propio de una manera mucho mas sencilla por medio de JOINS y FUNTION.

JOINS:

La sentencia JOIN (unir, combinar) en SQL es una funcion la cual nos permite combinar registros de una o más tablas dentro de una base de datos. Dentro del lenguaje SQL podemos encontrar tres tipos de JOins: interno, externo y cruzado. Sin embargo el estándar ANSI del SQL especifica cinco tipos de JOIN: INNER, LEFT OUTER, RIGHT OUTER, FULL OUTER y CROSS. Donde vemos que con estos, vemos que una tabla puede unirse a sí misma, produciendo una auto-combinación, SELF-JOIN. O podemos aplicar in INNER JOIN para obtener una tabla cullo producto sea en donde de dos tablas se seleccionan los elementos de ambas tablas en donde tengan un valor en comun dentro de uno de los campos.

La estructura de los JOINS se puede encontrar de la manera como se halla en la imagen de la parte superior, en donde vemos que inicialmente se selecciona una tabla por medio de la funcion SELECT, aplican las condiciones respectivas y luego se selecciona el tipo de JOIN junto con la otra tabla seleccionada de manera similar.

En el siguiente ejemplo podremos ver aquellos juegos cuyo ID sea el mismo que el de alguno de los usuarios:

SELECT * from clientes Inner JOIN juegos ON clientes.ID=juegos.ID;

O podremos asignales a todos los desarrolladores los mismos juegos:

SELECT * from desarrolladores Cross JOIN juegos;

Consideramos que con los ejemplos se puede ver como es que debe de ser el uso de los JOINS mostrandolos de manera sencilla y especificando la funcion que cumplen dentro de las selecciones que estamos realizando.

FUNTION and PROCEDURE:

Una funcion es una función definida por el usuario la cual vemos que acepta ciertos parámetros y con estos realiza una acción, como un cálculo complejo que podria devuelvernos el resultado de esa acción como un valor. El valor de retorno puede ser un valor escalar (único) o una tabla. Donde vemos que este puede ser utilizado con las siguientes formas:

1.En instrucciones Transact-SQL como SELECT.

2.Para definir una columna en una tabla.

3.Para definir una restricción CHECK en una columna.

4.Para reemplazar un procedimiento almacenado.


En el siguiente ejemplo veremos como actualizar la tabla de ofertas, borrando todas las ofertas mostradas:

Create or REPLACE PROCEDURE vaciar_tabla()

LANGUAGE 'plpgsql'

as $$

BEGIN

DELETE FROM Ofertas;

END;

$$;

Y para hacer uso de este procedimiento lo unico que debemos de hacer es usar el comando CALL vaciar_tabla()

Ahora para mostrar que son las funciones mostraremos como retornar una tabla de promedios de precio por categorias:

Create or REPLACE FUNCTION AVG_tabla()

RETURNS table(cat VARCHAR(90),promedio numeric(10,2))

LANGUAGE 'plpgsql'

as $$

BEGIN

RETURN QUERY

SELECT categoria, AVG(precio)

from categoria_juegos INNER JOIN juegos

ON categoria_juegos.id_juego =juegos.id

GROUP BY categoria;

END;

$$;

Y podremos invocar a este metodo utilizando un simple select:

SELECT * FROM AVG_tabla()

Ranking:

La funcion para rankear nos ayuda a visualizar por puestos los datos que quisieramos manipular. Con el siguiente ejemplo quisieramos mostrar los juegos que son mas jugados y clasificarlos por un rankeamiento.

SELECT ID_juego, sum(extract (hour from(hora_final - hora_inicio))) as playtime, rank() over(order by sum(extract (hour from(hora_final - hora_inicio))) DESC) from jugando where hora_inicio BETWEEN now()- Interval '7 day' and now() Group by ID_juego;
